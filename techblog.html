D3.js: Bringing Data to Life

Bryan Yan

How I Discovered D3
Recently I've been using D3 as part of my work at the Center of Machine Learning and Health. I've been working on parsing data produced by GenAmap to render a matrix view and dendrograph using d3.js with zoom in/out capabilities, a minimap, and a gradient according to correlation values. Hovering over the data points shows all the pertinent data, and you can select and save different subsets. It's really an amazing library with so many insane features. Even after using it so much, I feel like I have so much to learn about it and have barely even scratched the surface of what it's capable of.


What is D3?
D3.js (Data Driven Documents) is a Javascript library to produce dynamic, interactive data visualizations in web browsers. Written by Mike Bostock (who is a god), it allows you to attach data to the DOM and through usage of CSS and different features of D3, you can make this data interactive with different transformations and transitions.
D3 is used in basically every NYTimes graphic. Recently there's been a lot of cool ones because of the elections. It's pretty dope.

How do I Get Started?
D3 is an insanely vast library and there's a lot to learn about it. The good news is, there are plenty of different resources for getting started with D3. In addition, D3 has these things called "blocks" where different users will create small demos with the code needed for the display.
